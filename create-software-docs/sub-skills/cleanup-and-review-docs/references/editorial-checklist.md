# Editorial Checklist

## Purpose

Use this checklist after validation and before final delivery to confirm editorial consistency.

## Applies To

- `cleanup-and-review-docs`

## Checklist

### Editorial Consistency

- [ ] headings use a consistent hierarchy
- [ ] similar concepts use the same names across docs
- [ ] lists and tables follow a consistent style
- [ ] the same fact is not repeated unnecessarily in multiple files

### Readability

- [ ] long paragraphs are split when possible
- [ ] repetitive wording is reduced
- [ ] explanations are concise but still evidence-preserving
- [ ] cross-references help the reader navigate instead of duplicating content

### Hygiene

- [ ] no stale placeholders remain
- [ ] `Inferred` and `Needs confirmation` are used consistently
- [ ] no obvious formatting artifacts remain
- [ ] minor broken links or path mistakes are fixed

### Final Review

- [ ] the docs look ready to share with another engineer or LLM
- [ ] no cleanup change introduced new unsupported claims
- [ ] a final validation pass is requested if cleanup made substantial edits

## Usage Notes

- use this as a finishing checklist, not as a source of new content
