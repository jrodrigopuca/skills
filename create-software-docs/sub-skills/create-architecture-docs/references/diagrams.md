# Architecture Diagram Rules

## Purpose

Constrain architecture diagrams to the selected `{scope}` and keep them aligned with repository evidence.

## Applies To

- `create-architecture-docs`

## Rules

- use the shared guidance in `../../references/diagrams.md`
- diagrams must reflect the selected `{scope}`, not the whole repo unless the scope is the repo root
- omit diagrams that would be speculative
- prefer real module and service names over placeholders
- keep diagram scope consistent with the text narrative

## Constraints

- do not widen diagram scope beyond the selected target without explicitly stating it
