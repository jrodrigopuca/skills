# Validation Output Checklist

## Purpose

Define the minimum structure of the validation summary returned to the orchestrator.

## Applies To

- `validate-generated-docs`

## Checklist

- [ ] generated files checked
- [ ] files outside scope, if any
- [ ] broken links, if any
- [ ] missing evidence notes, if any
- [ ] speculative or unresolved sections, if any
- [ ] Mermaid inconsistencies, if any
- [ ] final pass/fail summary

## Usage Notes

- include empty categories explicitly when nothing was found if that helps the orchestrator summarize results
