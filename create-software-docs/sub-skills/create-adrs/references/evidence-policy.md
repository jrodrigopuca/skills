# ADR Evidence Policy

## Purpose

Keep ADR rationale honest about what is directly evidenced versus reconstructed from repository state.

## Applies To

- `create-adrs`

## Rules

- **Direct evidence**: supported by config, code, docs, issues, or commits
- **Reconstructed**: inferred from repository state and architectural patterns
- **Needs confirmation**: not sufficiently evidenced

## Constraints

- do not present reconstructed rationale as if it were directly documented by the team
